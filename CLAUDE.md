## Style

- Use `LOGGER` from `com.mojang.logging.LogUtils`.
- Prefer dedicated tools (`Read`, `Edit`, `Glob`, `Grep`) over shell `cat` / `grep` / `sed`.
- Comments: minimal. Only when WHY is non-obvious (a hidden constraint, a workaround for a specific bug, behavior that would surprise a reader). Do not narrate WHAT the code does.
- Public method signatures are stable; deprecate before removing.
- Don't add features, refactor, or introduce abstractions beyond what the task requires.
- Don't add error handling, fallbacks, or validation for scenarios that can't happen.
